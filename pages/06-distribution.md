# Дистрибуция в командах

<div class="mt-6 text-lg">

Какие артефакты нужны:

- **рулы / `agents.md`** – как устроен ваш проект
- **скилл** – ревью кода, работа с внутренними инструментами
- **MCP** для доступа к БД

</div>

<div class="mt-6 text-xl">
<strong class="accent-2">Проблема:</strong> каждый настраивает агента по-своему – кто-то не настраивает вовсе, кто-то городит своё, кто-то делает неэффективно. Опыт у всех <strong>разный</strong>, а хочется <strong class="accent">единого</strong> – в команде или под конкретный сценарий
</div>

<div class="mt-4 text-xl">

Единый репозиторий артефактов — одно место для:

</div>

<div class="chips mt-4">

  <span class="chip on">skills</span>
  <span class="chip on">modes</span>

<div class="mt-6 text-lg">

Зачем:

- **простое дискавери** артефактов

- **контроль актуальности** — если где-то есть ссылка на артефакт, отслеживаем валидность в CI

</div>

# Пресеты

<div class="grid grid-cols-2 gap-8 mt-6">

<div class="space-y-4 text-lg">

<div>

**Пресет** — yaml-файл с набором нужных артефактов под команду или сценарий

</div>


<div>

Обновили пресет → <strong class="accent">вся команда подтянула</strong> изменения при следующем запуске

</div>

</div>

<div>

```yaml
name: "YCA"
description: >
  Правила для разработки Code Assistant
rules:
  - yca/*
skills:
  - skills/arc
mcp:
  - devtools
  - tracker_mcp
  - wiki
```

</div>

</div>

<!--
Пресет как линтер: один раз настроил для команды — дальше все пользуются и автоматически получают обновления.
-->

# Как пресет попадает к разработчику

<div class="dlv-grid mt-6">

  <div class="ms-box ms-box--mcp dlv-box">
    <div class="ms-box__h">Внутренний агент</div>
    <div class="ms-box__sub">наши IDE и CLI</div>
    <ul>
      <li>в начале диалога <strong>сам обнаруживает</strong> подходящий пресет</li>
      <li>предлагает поставить — <strong>в один клик</strong></li>
      <li><em>человек ничего не настраивает</em></li>
    </ul>
  </div>

  <div class="ms-box ms-box--skill dlv-box" v-click>
    <div class="ms-box__h">Внешний инструмент</div>
    <div class="ms-box__sub">OpenCode, Pi и др.</div>
    <ul>
      <li>один раз ставишь <strong>хук</strong> на запуск агента</li>
      <li>хук дёргает нашу утилиту — она <strong>ставит, проверяет и обновляет</strong> артефакты</li>
      <li><em>настроил один раз — забыл</em></li>
    </ul>
  </div>

</div>

<div class="dlv-out" v-click>
Обновили пресет в репозитории → у всей команды подтянулось <strong class="accent">при следующем запуске</strong>
</div>

<!--
Внутри — push: агент сам находит пресет и предлагает. Снаружи — pull через хук: один раз настроил, дальше утилита всё сама.
-->

# А если у вас не зоопарк, а один агент — плагины

<div class="mt-4 text-xl">

Пресеты нужны не всем. Если вся команда сидит на <strong>одном агенте</strong> — городить своё не надо, есть готовый механизм: **плагины**

</div>

<div class="mt-6 text-lg">

- <strong class="accent">Плагин</strong> — бандл из скиллов, команд, MCP и интеграций
- ставится из <strong class="accent">маркетплейса агента</strong>
- механизм уже есть в современных агентах из коробки

</div>

<!--
Не привязываемся к конкретным инструментам — плагины как механизм есть у современных агентов.
Пресеты нужны, когда зоопарк инструментов; на одном агенте достаточно плагинов.
-->
